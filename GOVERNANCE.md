# Governance

`wimsey` is an open, vendor-neutral project. This document describes how it is
run. It is intentionally lightweight while the project is young and will grow
toward the CNCF project governance expectations as the maintainer base widens.

## Roles

- **Contributors** open issues and pull requests. Anyone can be a contributor.
- **Maintainers** review and merge changes, triage issues, and cut releases.
  Maintainers are listed in [`MAINTAINERS.md`](MAINTAINERS.md).

## Decision making

- Routine changes are approved by maintainer review of a pull request.
- Significant changes (spec revision bumps, new crates, governance changes) are
  proposed as an issue or pull request and decided by maintainer consensus.
- The project explicitly seeks maintainers from more than one organisation to
  preserve vendor neutrality.

## Adding maintainers

A contributor with a sustained track record of quality contributions may be
nominated by an existing maintainer and confirmed by maintainer consensus.

## Changing this document

Changes to governance follow the significant-change process above.
